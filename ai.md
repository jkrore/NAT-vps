第 1 梯队：最重要，天天用
1. 初始化 Git 项目
把项目初始化成一个 Git 工程，注意排除不需要的文件。
2. 我不会 Git，让它代管
我不会 Git，你帮我安全处理分支、提交和 PR，不要直接覆盖 main。
3. 生成忽略文件
根据当前项目生成合适的 .gitignore，不要改其他文件。
4. 检查不该提交的东西
检查项目里哪些文件不应该提交到 Git，并帮我加入 .gitignore。
5. 第一次整理并提交
整理当前项目并做第一次 Git 提交，注意不要提交敏感文件和垃圾文件。
6. 新分支开发
新建一个功能分支完成这个需求，不要直接改 main。
7. 改完创建 PR
改完后创建 PR，不要直接合并。
8. 修报错
根据这个报错修复问题，只做最小修改。
9. 先分析，别急着改
先帮我定位这个问题的根因，不要急着改代码。
10. 生成 README 和 AGENTS
根据真实项目生成 README.md 和 AGENTS.md，不要编造功能。
第 2 梯队：GitHub / PR 常用

Codex Web 连接 GitHub 后，可以读取仓库并从工作结果创建 Pull Request，所以你不需要自己会 Git 命令。

11. 让 Codex 处理仓库
在这个 GitHub 仓库里完成这个需求，改完后创建 PR，不要直接合并。
12. 检查仓库
帮我检查这个仓库当前状态，告诉我有没有问题，不要修改。
13. 审查 PR
帮我审查这个 PR，重点看 bug、安全风险和不该提交的文件。
14. 看 PR 能不能合并
这个 PR 能不能合并？帮我检查风险。
15. 合并分支
把当前分支安全合并回主分支，有冲突先告诉我。
16. 写 PR 描述
把这次改动整理成一个 PR，标题和描述都帮我写好。
第 3 梯队：浏览器自动操作 / 自动登录

Codex 的 Computer Use 可以看屏幕、点击、输入，适合网页或桌面软件操作；但涉及登录、验证码、密钥、付款、删除这类敏感操作，最好让它停下，你自己处理。

17. 浏览器万能句
用浏览器帮我完成这个操作，需要登录、验证码、密钥、付款、删除时停下让我自己处理。
18. 登录我自己来
打开这个网站，需要登录时停下，我自己登录，登录后你继续操作。
19. GitHub 网页操作
打开 GitHub 帮我操作这个仓库，需要登录时我自己登录，不要删除或合并任何东西。
20. 部署网页操作
用浏览器帮我部署这个项目，需要登录和填写密钥时停下让我自己操作。
21. 看网页报错
打开网页复现这个问题，找到原因并修复，改完再测试一遍。
第 4 梯队：文档 / AGENTS.md

AGENTS.md 很重要，它相当于“写给 AI 的 README”。官方说 Codex 开始工作前会读取 AGENTS.md；AGENTS.md 社区也把它定义成给 coding agents 的项目规则文件。

22. 生成 AGENTS.md
根据当前项目生成 AGENTS.md，让以后 AI 改代码时知道项目规则。
23. 更新 AGENTS.md
更新 AGENTS.md，加入项目运行方式、测试方式和不要乱改文件的规则。
24. 生成 README
根据项目真实代码生成 README.md，不要编造不存在的功能。
25. 更新 README
根据当前项目更新 README.md，写清楚安装、运行和配置方法。
26. 生成新手说明
生成一份新手能看懂的使用说明，放到 docs 目录。
第 5 梯队：修 bug / 加功能
27. 修 bug 最常用
根据这个报错修复问题，只做最小修改。
28. 先找原因
先帮我定位这个问题的根因，不要急着改代码。
29. 改完说明
修完后告诉我原因、改了哪些文件、怎么验证。
30. 添加功能
给项目添加这个功能，保持现有功能不变。
31. 最小改动
按最小改动实现这个功能，不要大规模重构。
32. 先给方案
先看项目结构，给我一个实现方案，暂时不要改代码。
第 6 梯队：项目体检 / 清理
33. 项目体检
帮我体检这个项目，找出结构、依赖、安全和文档问题，不要修改。
34. 清理垃圾
帮我清理项目里的临时文件、缓存和构建产物，不要删源码。
35. 检查敏感信息
检查项目里有没有 token、密码、密钥、cookie，不要打印完整内容。
36. 检查依赖
检查项目依赖是否有明显无用、重复或安全风险。
37. 检查提交风险
提交前帮我检查有没有敏感信息、缓存、日志和依赖目录。
第 7 梯队：运行 / 部署 / 一键脚本
38. 跑通项目
帮我把项目跑通，缺什么依赖就告诉我或自动补齐。
39. 找运行方法
根据项目文件判断怎么运行，并告诉我命令。
40. 生成一键启动脚本
给项目生成一键启动脚本，Windows 和 Linux 都尽量支持。
41. Docker
根据项目生成 Dockerfile 和 .dockerignore，注意不要打包敏感文件。
42. 修部署失败
帮我修复部署失败的问题，找到原因并给出可运行方案。
